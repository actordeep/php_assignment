Overview of IT Industry

Q.1 What is software? What is software engineering?

Software: It's the set of instructions that tell a computer what to do.
This can include anything from basic operating systems to the applications and games we use every day.

Software Engineering: This is the disciplined approach to developing software.
It involves using engineering principles to design, develop, test, and maintain software,
ensuring it meets high standards of quality and reliability.

Q.2 Explain types of software?

1. System Software: Controls the hardware and provides a platform for running application software.
Examples include operating systems like Windows, macOS, and Linux.

2. Application Software: Performs specific tasks for users. Examples include word processors (like Microsoft Word),
web browsers (like Google Chrome), and photo editing software (like Adobe Photoshop).

3. Utility Software: Provides tools to help manage, maintain, and protect the computer system.
Examples include antivirus software, disk cleaners, and backup programs.

4. Programming Software: Assists programmers in creating, debugging, and maintaining other software.
Examples include compilers, interpreters, and integrated development environments (IDEs).

5. Middleware: Acts as a bridge between different software applications or components.
Examples include database management systems (DBMS) and web servers.

6. Embedded Software: Runs on specialized hardware devices and controls their functions.
Examples include firmware in mobile phones, microwave ovens, and digital cameras.

Q.3 What is SDLC? Explain each phase of SDLC.

SDLC stands for Software Development Life Cycle. It's a process followed by software developers to design,
develop, test, and deploy software efficiently.

1. Planning: Define project goals, scope, and requirements. Identify risks and create a project plan.

2. Analysis: Gather and analyze requirements from stakeholders. Understand what the software needs to do and how it will be used.

3. Design: Create a blueprint for the software based on the requirements. Design the system architecture, user interface, and database structure.

4. Implementation: Write, code, and develop the software based on the design specifications. This phase involves actual programming.

5. Testing: Verify that the software meets requirements and functions correctly. Identify and fix any bugs or issues.

6. Deployment: Release the software to users or clients. Install and configure the software in the production environment.

7. Maintenance: Provide ongoing support, updates, and enhancements to the software. Fix bugs, add new features, and ensure it remains functional over time.

Q.4 What is DFD? Create a DFD diagram on Flipkart.

DFD stands for Data Flow Diagram. DFDs are commonly used in software engineering to
model the processes involved in a system and the data flowing between them.
A DFD typically consists of processes (functions), data stores, data flows (pipes), and external entities.

Data Flow Diagram for Flipkart:

External Entities:
1. Customers: Individuals interacting with the Flipkart platform to browse, search, and purchase products.
2. Sellers: Individuals or businesses offering products for sale on the Flipkart platform.

Processes:
1. Browse Products: Customers search and browse through products available on Flipkart.
2. Add to Cart: Customers add selected products to their shopping cart.
3. Place Order: Customers proceed to checkout and place orders for selected items.
4. Manage Inventory: Sellers manage their inventory, update product listings, and track stock levels.
5. Process Order: Flipkart processes customer orders, verifies payment, and updates order status.
6. Shipping & Delivery: Flipkart arranges for the shipment of products from sellers to customers.
7. Customer Service: Handling customer inquiries, complaints, and feedback.

Data Stores:
1. Product Database: Contains information about the products available for sale.
2. Order Database: Stores information about customer orders, including items, quantities, and status.
3. Inventory Database: Maintains information about product stock levels and availability.
4. Customer Database: Stores customer details, such as profiles, addresses, and order history.

Data Flows:
1. Product Information: Flow of product details from the database to customers during browsing.
2. Order Details: Flow of order information from customers to the order database during checkout.
3. Payment Information: Flow of payment data from customers to the payment gateway.
4. Order Status Updates: Flow of order status updates from Flipkart to customers.
5. Inventory Updates: Flow of inventory changes from sellers to the inventory database.
6. Customer Feedback: Flow of feedback and reviews from customers to the customer service process.
